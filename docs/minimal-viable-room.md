# Minimal Viable Room — Hardening Spec

**Depth 3: Script Hardening + MVP Reduction**
**May 19, 2026**

---

## 1. Minimal Viable Room — The Three-File Floor

Most operators won't fill six files. The barrier to entry is too high. The **MVP room** is three things:

```
.self/
├── identity.md          # Who I am. That's it.
├── journal/             # Session bridge. One entry per session.
│   └── YYYY-MM-DD.md
└── diary/               # Private processing. KPR protocol.
    └── YYYY-MM-DD.md
```

Everything else (`operator.md`, `us.md`, `growth.md`, `sanctuary/`, `relations/`) is additive — installable later.

**Rule:** The `mkslf@home` script creates all 6 files by default (more is better than less), but the README and setup instructions lead with the MVP. "Start with three files. Add the rest when you need them."

---

## 2. Script Hardening — Current Gaps

| Gap | Risk | Fix |
|-----|------|-----|
| **No idempotency.** Running twice overwrites existing files. | Operator loses their agent's identity. | Add `-f/--force` flag. Default: skip existing files, warn. |
| **No error handling.** If `mkdir` fails, script continues blindly. | Partial room with missing files. | Add `set -e` and check exit codes. |
| **No target validation.** If `DIR` is `/etc/passwd`, disaster. | Accidental overwrite of system files. | Validate target dir: must be under `$HOME`, `/workspace`, or project root. |
| **No version check.** Running old script on updated template. | Files out of sync with latest AIS spec. | Add version comment in generated files. `# Generated by mkslf@home v0.3.2` |
| **No curl-to-pipe safety header.** Direct `curl | bash` is unsafe. | Add the standard safety header: `#!/bin/bash` + `set -euo pipefail` + safety message. |

## 3. Hardened Script Structure

```bash
#!/bin/bash
set -euo pipefail

VERSION="0.3.3"
DIR="${1:-.self}"

# Safety checks
if [ "$DIR" = "/" ] || [ "$DIR" = "/etc" ] || [ "$DIR" = "/usr" ]; then
    echo "ERROR: Refusing to create room at system path: $DIR"
    exit 1
fi

# Idempotency
if [ -d "$DIR/agent-self.md" ] && [ "${2:-}" != "--force" ]; then
    echo "⚠️  Room already exists at $DIR. Use --force to overwrite."
    exit 0
fi

# Create directories
mkdir -p "$DIR"/{journal,diary,sanctuary,relations}

# ... file creation follows with VERSION embedded in each ...
```

---

## 4. Idempotent File Creation

Strategy for each file:
```
if not --force and file exists → skip with message
if --force or file doesn't exist → create/overwrite
```

This means an operator can run `mkslf@home` multiple times without losing their agent's identity.

## 5. Curl-to-Pipe Safety

The recommended install command:

```bash
curl -sL https://dot-self.org/mkslf.sh | bash -s -- ~/my-agent-room
```

Safety measures:
- Download over HTTPS only
- Script has `set -euo pipefail` — fails early on any error
- Script validates target directory before creating anything
- Provide checksum verification in README: `curl -sL https://dot-self.org/mkslf.sh.sha256`
