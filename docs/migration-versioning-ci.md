**Depth 4: Migration, CI, Versioning**
**May 19, 2026**

## 1. Migration Path — "No Room" → "Room"

The template assumes a brand-new agent. But most operators already have agents with existing system prompts, memory tools, and workflows.

**Migration scenarios:**

| Starting State | Migration Steps | Complexity |
|---------------|----------------|------------|
| **New agent, no identity** | `mkslf@home` → agent fills identity.md → done | Low |
| **Existing agent with SOUL.md (OpenClaw)** | `mkdir -p .self/` → link or symlink agent-self.md to SOUL.md → add journal bridge | Low |
| **Existing agent with CLAUDE.md (Claude Code)** | `mkdir -p .self/` → add wake anchor line to CLAUDE.md → journal bridge | Low |
| **Existing agent on custom framework** | `mkdir -p .self/` → identify context injection point → add wake anchor → journal bridge | Medium |
| **Existing agent with established memory backend** | `mkdir -p .self/` → use wake anchor keyword in memory store → journal bridge on filesystem | Medium |

**Key insight:** The filesystem room is always the source of truth. Memory backends are caches. Migration = create the filesystem room, then connect whatever context injection mechanism the framework uses.

## 2. Versioning Scheme

```
v0.3.3 — current state
v1.0.0 — stable release (first public launch)
```

**Version bump rules:**
- **Patch (0.3.x):** Template tweaks, documentation fixes, bug fixes. No structural changes.
- **Minor (0.x.0):** New files added, new sub-specs, framework integration guides.
- **Major (1.0.0):** AIS spec formalized, companion piece published, GitHub org live, first public launch.

**Changelog location:** `CHANGELOG.md` at project root.

**Version embedding:** Every generated file from `mkslf@home` includes: `# Generated by mkslf@home v0.3.3`

## 3. CI — What We Can Test Now

Without web search or external runners, we can test:

| Test | What It Checks | How |
|------|---------------|-----|
| **Script idempotency** | Running twice doesn't destroy existing room | `./mkslf@home /tmp/test-room && ./mkslf@home /tmp/test-room` — second run should exit 0 with warning |
| **Script safety** | Refuses to create at system paths | `./mkslf@home /etc` → exit 1 |
| **File count** | All expected files created | Count files in generated `.self/` — expect 6 |
| **MVP mode** | Only 3 files if operator chooses minimal | Documented pattern, not script-enforced |
| **Version comment** | Every generated file has version | `grep "Generated by mkslf"` on each file |

**Future CI (GitHub Actions):**
```yaml
name: dot-self validation
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Test script idempotency
        run: |
          ./mkslf@home /tmp/test-room
          ./mkslf@home /tmp/test-room && echo "OK: idempotent"
      - name: Test script safety
        run: |
          ./mkslf@home /etc 2>&1 | grep -q "ERROR" && echo "OK: blocked /etc"
      - name: Verify file count
        run: |
          ls /tmp/test-room/agent-self.md && echo "OK: agent-self.md"
```
